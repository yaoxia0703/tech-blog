# システムアーキテクチャ

本ページでは、個人開発プロジェクトにおける中核的なアーキテクチャ設計について紹介します。

本システムは **Spring Boot 3** を用いて構築しており、ドメイン駆動設計（DDD）の考え方を取り入れた **モジュラー型バックエンドアーキテクチャ** を採用しています。  
また、**RBAC 認可**、**JWT 認証**、および **明確なモジュール分離** を重視して設計しています。

## 全体アーキテクチャ

本プロジェクトは **フロントエンド・バックエンド分離アーキテクチャ** を採用しています。

バックエンド  
Spring Boot REST API

フロントエンド  
Vue 3 + Element Plus

認証  
JWT

認可  
RBAC

データベース  
MySQL

キャッシュ  
Redis

## システム構成

フロントエンド（Vue 3 + Element Plus）  
│  
│ HTTP / REST API  
▼  
バックエンド（Spring Boot）  
│  
├── Security（Spring Security + JWT）  
├── RBAC 権限管理システム  
├── コンテンツ管理  
│  
▼  
データベース（MySQL） + キャッシュ（Redis）

## RBAC 認可

本システムでは、ユーザー権限を管理するために **RBAC（Role Based Access Control）** を採用しています。

各ユーザーには 1 つ以上のロールを割り当てることができ、各ロールにはシステム利用を制御する権限セットが紐づきます。

## 主要テーブル

- user
- role
- permission
- menu
- role_permission
- role_menu

## 認証フロー

1. ユーザーがログイン情報を送信する
2. バックエンドがユーザー名とパスワードを検証する
3. サーバーが JWT トークンを生成する
4. クライアントがトークンを保存する
5. 以後の各リクエストにトークンを付与する
6. Spring Security がトークンを検証し、ユーザー情報を復元する

## メリット

- ステートレス認証を実現できる
- サーバー側のセッション保存負荷を削減できる
- 分散システムとの相性が良い

## バックエンドモジュール設計

バックエンドは、保守性と責務分離を高めるために、**マルチモジュール構成** で設計しています。

## 主なモジュール

**blog-common**  
共通ユーティリティおよび共有コンポーネントを提供するモジュール。

**blog-framework**  
Security 設定、JWT 認証、グローバル例外処理などを提供する共通基盤モジュール。

**blog-module-system**  
ユーザー、ロール、権限、メニューなどを含む RBAC システム領域を担当するモジュール。

**blog-module-content**  
記事、カテゴリ、タグ、コメントなど、ブログのコンテンツ領域を担当するモジュール。

**blog-admin-api**  
管理画面向けバックエンド API を提供するモジュール。

**blog-portal-api**  
ブログのフロント画面向け公開 API を提供するモジュール。

## API 設計方針

バックエンド API は **RESTful 設計原則** に基づいて設計しています。

## 主な設計ポイント

- 統一レスポンス構造（`Result<T>`）
- 一貫した例外・エラーハンドリング
- API の明確な分離（Admin APIs / Portal APIs）
- 標準化されたリクエスト / レスポンス形式

## アーキテクチャ設計目標

本アーキテクチャは、以下の目標を重視しています。

- 明確なモジュール分離
- 拡張性の高いバックエンド設計
- 安全な認証・認可
- 保守しやすいコード構造
